Returns a new unlimited thread pool with zero core thread count to make sure no threads are created by default, KEEP_ALIVE_TIME_MS keep alive time, the DEFAULT_SOURCE_UNLIMITED_EXECUTOR_NAME thread name prefix, the DEFAULT uncaught throwable strategy, and the SynchronousQueue since using default unbounded blocking queue, for example, PriorityBlockingQueue effectively won't create more than corePoolSize threads. See ThreadPoolExecutor documentation.

Source executors allow network operations on their threads.
